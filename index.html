White Paper

Hash Humanity is designed to verify unique human participation in digital communities without collecting identity data or relying on behavioral profiling. The system uses privacy-preserving proofs from World ID and validates them against the public registry state.

Red, white, black trust layout World ID proof validation Merkle membership integrity

World ID Verification Flow

Hash Humanity verifies uniqueness using World ID zero-knowledge proofs. A user does not transmit identity or biometric information. Instead, the system validates a proof of membership in the verified-human set, and uses a nullifier to prevent reuse for protected actions.

Figure 1. The user generates a privacy-preserving proof inside World App. Hash Humanity verifies the proof against the World ID registry state without receiving identity data.
  • Proof: Demonstrates membership in the verified set without revealing which identity the user is.
  • Merkle root: Compact fingerprint of the full verified-human registry.
  • Nullifier: Prevents the same credential from being reused for a protected action.

Merkle Tree Membership and Proof Validation

World ID represents the verified-human registry as a Merkle tree. Each verified credential corresponds to a hashed leaf. A short proof allows a verifier to recompute the root and confirm membership without revealing which leaf belongs to the user.

Figure 2. Merkle tree membership. The Merkle root commits to the entire registry. A proof shows membership in the set without revealing which credential is used.
  • Integrity: any change to a leaf changes the root.
  • Efficiency: proofs are compact and verification remains fast at large scale.
  • Privacy: the verifier learns only that the user is a member, not which member.
